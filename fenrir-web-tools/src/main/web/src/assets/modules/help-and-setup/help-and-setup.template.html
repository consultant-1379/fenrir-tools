    •      Tunnel setup for FLS queries An SSH tunnel needs to be established towards the FLS database to have access to the reports generated in the option 'File Collection Verification'
    The port 5432 must be available on your computer.

    For Windows users:

    If you're connecting to a physical server:

    ssh -L 5432:127.0.0.1:2500 [MS_SERVER_USERNAME]@[MS_SERVER_HOST_OR_IP] [sudo] ssh -L 2500:localhost:5432 litp-admin@db-1

    See 'Notes' below.

    If you're connecting to a vApp:

    ssh -L 5432:127.0.0.1:2500 root@192.168.0.42 ssh -L 2500:localhost:5432 litp-admin@db-1

    For Linux users:

    If you're connecting to a physical server:

    ssh -o UserKnownHostsFile=/dev/null -o StrictHostKeyChecking=no -t -L 5432:127.0.0.1:2500[MS_SERVER_USERNAME]@[MS_SERVER_HOST_OR_IP] [sudo] ssh -o UserKnownHostsFile=/dev/null -o StrictHostKeyChecking=no -L 2500:localhost:5432 litp-admin@db-1

    See 'Notes' below.

    If you're connecting to a vApp:

    ssh -o UserKnownHostsFile=/dev/null -o StrictHostKeyChecking=no -t -L 5432:127.0.0.1:2500 root@atvts[VAPP_NUMBER].athtem.eei.ericsson.se ssh -o UserKnownHostsFile=/dev/null -o StrictHostKeyChecking=no -t -L 2500:127.0.0.1:2500 root@ms-1 ssh -o UserKnownHostsFile=/dev/null -o StrictHostKeyChecking=no -L 2500:127.0.0.1:5432 litp-admin@db-1

    Notes:

    [VAPP_NUMBER]: Replace for the vapp number from your booking (atvtsXXXX.athtem.eei.ericsson.se)
    [MS_SERVER_USERNAME]: Replace for user name for the MS server
    [MS_SERVER_HOST_OR_IP]: Replace for the IP or hostname of the MS server
    [sudo]: Optional. You may need to include this bit if the user name for the MS server is anything other than 'root' (usually on S2, S4 bookings)
    db-1: Sometimes PostgreSQL is not running on db-1. If you get errors like 'Connection refused', you can try using db-2 instead

    Customising runtime configuration for this tool To change the HTTP port this tool listens on, launch the application using the following command:

    $ java -Dserver.port=9080 -jar fenrir-tools.jar
